# План написания уроков mmBasic с проверкой на устройстве

Этот файл — внутренняя процедура, не часть курса. Описывает, как писать новые уроки или править существующие, проверяя каждый нетривиальный сниппет на реальном PicoCalc через USB-C.

## Подключение к PicoCalc

- **Кабель в USB-C на корпусе** PicoCalc (не в micro-USB самого Pico — там REPL не идёт).
  - Кабель должен быть data-grade. Charge-only не покажет нового устройства в `/dev/`.
- **Порт на macOS**: что-то вида `/dev/cu.usbserial-NNN` (STM32-мост клавиатуры пробрасывает UART REPL на USB-CDC). Точные цифры зависят от слота USB-A на хосте — `cu.usbserial-10`, `cu.usbserial-210`, и т.п.
- **Скорость**: 115200, 8N1, no flow control.
- **Хелпер**: `.tools/picomite_repl.py` сам сканирует `/dev/cu.usbserial-*` и берёт единственное совпадение. Если их несколько — печатает кандидатов и просит указать `--port` или переменную окружения `PICOMITE_PORT`.

Запуск:
```
cd mmBasic
uv run --with pyserial .tools/picomite_repl.py -c 'PRINT MM.VER'
```

`uv` создаёт venv с `pyserial` на лету, ничего не остаётся в системе.

## Процедура написания урока

Для каждого нового урока:

1. **План уроков** — в `README.md` есть таблица. Уточни у пользователя, что в фокусе.
2. **Pre-flight checks** — перед написанием прогоняешь через helper все нетривиальные команды, которые попадут в урок. Если PicoMite ругается — найди работающий вариант (попробуй `HELP <cmd>` через мануал онлайн, варианты синтаксиса по форумам, фактическое поведение).
3. **Пишешь урок** в `mmBasic/lessons/NN-name.md`. Тон — «ты уже знал QBasic». Не разжёвывать базу.
4. **После записи** — повторно прогоняешь все сниппеты урока через helper. Если что-то добавлено/изменено по ходу — может оказаться непроверенным.
5. **Интерактивные программы** (с `INPUT`, `INKEY$` в цикле) проверяешь только по парсингу: загружаешь через `AUTOSAVE` или поэлементно, делаешь `LIST`, убеждаешься в отсутствии syntax error. Реальное поведение проверит пользователь.
6. **Обновляешь** таблицу уроков в `README.md` и `CHANGELOG.md` (буллет под сегодняшней датой).
7. **Останавливаешься после каждого урока**, ждёшь ревью пользователя, прежде чем идти дальше.

## Gotchas (накопленный список)

Эти штуки заводская прошивка PicoCalc делает не так, как описывает «абстрактный» PicoMite Manual. По мере прохождения курса список будет пополняться.

1. **`CHDIR "B:/..."` из `A:`** → `Error: only valid on current drive`.
   - Сначала отдельной командой переключи диск (`B:` или `A:`), потом `CHDIR "путь_внутри_диска"`.
2. **`Home` × 2 для terminate edit** — на PicoCalc не работает.
   - Клавиатура компактная, `Home`/`End`/`Insert` либо через `Fn`-комбо, либо вообще не передаются как ожидаемые скан-коды. Не закладываться.
3. **`HELP <команда>`** → `Error: A:/help.txt not found`.
   - Заводская прошивка не положила help.txt в LittleFS. Можно не доустанавливать — проще лезть в онлайн-мануал.
4. **`KEYDOWN(...)` не объявлено** в этой прошивке.
   - В мануале PicoMite описано как функция для USB-keyboard. На PicoCalc клавиатура через I²C-мост (см. `OPTION KEYBOARD I2C`) и `KEYDOWN` не реализован. Использовать только `INKEY$`.
5. **`OPTION EXPLICIT` + `FOR i = ...`** → `Error: I is not declared`.
   - Счётчик цикла должен быть `DIM`-нут заранее. В QBasic счётчики автоматические — здесь нет. Подсветить в уроке 02.
6. **`OPTION SERIAL CONSOLE DISABLE`** триггерит soft-reset, и настройка возвращается на дефолт `COM1, GP0, GP1`.
   - Прошивка ClockworkPi жёстко удерживает UART-консоль (она же — мост через STM32 для USB-C REPL). Не трогать. Подключаемся через USB-C, а не через Pico-USB.
7. **Многострочный `IF...ELSEIF...ENDIF` через `:`** не работает корректно. Однострочный IF (`IF cond THEN x` или с `ELSE`) — ок, но `ELSEIF`-цепочки требуют реальных переносов строк.
8. **`DO ... LOOP` через `:` в одной строке** ломает однострочный IF: `IF j=3 THEN EXIT DO : LOOP` — `LOOP` оказывается внутри блока THEN. Многострочно — без проблем.
9. **`OPTION BASE 1`** должно стоять **до первого `DIM` или `LOCAL`**, иначе `Error: Must be before DIM or LOCAL`.
10. **`FORMAT$` принимает только float-спецификаторы** (`%f`, `%g`, `%e`). `%d`, `%i`, `%,d`, `%05d` — все дают `Error: Illegal character in format specification`. Для целых — либо `STR$()`+`STRING$()`, либо `FORMAT$(n, "%07.0f")`.
11. **AUTOSAVE через serial-канал** с программой, содержащей `INPUT "..."`, ломается — REPL воспринимает `INPUT` как интерактивный prompt прямо в момент загрузки. Для проверки целиком надо использовать `OPEN ... FOR OUTPUT AS #1` + `PRINT #1, "..."` + `CLOSE`, потом `LOAD` (см. урок 07).
12. **`RENAME` синтаксис** — это `RENAME old$ AS new$`, **не `TO`**. `RENAME "a" TO "b"` → `Error : Syntax`.
13. **Директорийные операции и `RENAME` требуют «текущий диск»** — `RMDIR "B:/foo"` из `A:` → `Error : Only valid on current drive`, аналогично для `CHDIR` (см. п.1) и `RENAME` через границу дисков. В отличие от них, **`COPY` и `KILL` принимают абсолютные пути с префиксом любого диска** и работают кросс-дисково. Для move между `A:` и `B:` — `COPY` + `KILL`, не `RENAME`.
14. **Хоткеи в `EDIT` — задокументированный канон, но статус-строка их не показывает.** Внизу видно только `EDIT MODE  Ln: N  Col: N  INS`. По мануалу (Full Screen Editor):
    - **F1 = SAVE** — фиксирует текст редактора в программу в памяти и возвращает в обычный REPL `>`. Не запускает.
    - **F2 = RUN** — то же самое, но сразу запускает программу.
    - **Esc** — `Exit and discard all changes (Y/N): _`, выкидывает изменения. Это НЕ save&exit.
    - **F3** = FIND, **Shift-F3** = find next, **F4** = MARK (внутри редактора), **F5** = PASTE.
    - **Ctrl-Q / Ctrl-W** = альтернативы для F1 / F2 (для терминалов, не пробрасывающих Fn).
    - **F4 в REPL** (а не внутри редактора) — глобальный шорткат `EDIT`, поэтому связка «F1 → F4» = «сохранить и снова войти в редактор без запуска».
    - F1/F2 пишут в память, не в файл — чтобы получить файл, в REPL отдельно `SAVE "name.bas"`.
    - Источник правды: `references/manual/full_screen_editor.md` и `references/manual/first_steps.md`.
15. **`EDIT "name.bas"` для несуществующего файла** — открывает пустой редактор. Файл создаётся при выходе F1/F2. На существующем файле создаётся `.bak`-копия. См. `references/manual/edit_command.md`.
16. **У каждого диска своя cwd, и она запоминается между переключениями.** PicoMite держит две независимые «текущие папки» (одну для `A:`, одну для `B:`); команды `A:` / `B:` лишь переключают активную, не сбрасывают cwd. Поэтому после `B:` → `CHDIR "lessons"` → `A:` → `B:` ты снова в `B:/lessons`, а не в корне SD. Поведение из DOS/CP/M, не баг. Для возврата в корень — `CHDIR "/"` после переключения. Не забыть упомянуть в урок 07 (Файлы и SD).

## Перед уроками 04/06/07 нужно проверить

- **04 (Графика)**: `FRAMEBUFFER CREATE`, `FRAMEBUFFER WRITE F`, `FRAMEBUFFER COPY F, N`, `FRAMEBUFFER CLOSE`. Особенно `COPY` — критично для двойной буферизации.
- **05 (Ввод)**: что возвращает `INKEY$` при нажатии стрелок и спецклавиш PicoCalc. Это придётся снимать вручную: пользователь жмёт, я через helper читаю буфер. Делать в начале урока 05.
- **06 (Звук+время)**: `SETTICK period, target, n` с реальной SUB-функцией как обработчиком. Прерывание должно срабатывать асинхронно.
- **07 (Файлы)**: `OPEN ... FOR OUTPUT/INPUT/APPEND AS #1`, `PRINT #1`, `LINE INPUT #1`, `EOF`, `CLOSE`. Проверять и на `A:`, и на `B:`.
