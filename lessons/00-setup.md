# Урок 00 — Setup и REPL

Цель: освоиться в окружении PicoMite на PicoCalc — где я, как набирать команды, где живут программы, как редактировать и сохранять.

## Что у тебя сейчас

PicoCalc от ClockworkPi с предустановленной прошивкой:

```
PicoMite MMBasic RP2040 Edition v6.00.02 RC23
(Geoff Graham, Peter Mather)
```

Всё уже сконфигурировано производителем под этот корпус: 320×320 IPS-экран, аппаратная клавиатура (своя плата на STM32), I²S-звук на динамик, SD-слот, батарейка. Тебе не надо ничего флешить и крутить `OPTION SYSTEM SPI` — оно уже работает.

## REPL: где ты находишься

После загрузки видишь приглашение:

```
>
```

Это **REPL** (read-eval-print loop) — командная строка интерпретатора. Здесь два режима использования:

1. **Прямой режим** — пишешь команду, жмёшь Enter, она выполняется немедленно:

   ```basic
   > PRINT "Hello, PicoCalc!"
   Hello, PicoCalc!
   >
   ```

2. **Программный режим** — ты редактируешь *программу*, которая лежит в памяти, а потом её запускаешь командой `RUN`. Программу набирают в редакторе через `EDIT`.

Прерывание выполнения — **Ctrl-C** в любой момент.

## Команды, без которых никуда

| Команда | Что делает |
|---------|------------|
| `EDIT` | Открыть полноэкранный редактор текущей программы |
| `LIST` | Вывести текст программы в консоль |
| `RUN` | Запустить программу |
| `NEW` | Стереть программу из памяти (файлы на флеше не трогает) |
| `SAVE "name.bas"` | Сохранить текущую программу в файл |
| `LOAD "name.bas"` | Загрузить файл в память (как программу) |
| `FILES` | Показать содержимое текущей папки |
| `A:` / `B:` | Переключить **текущий диск** (это команда сама по себе) |
| `CHDIR "games"` | Сменить рабочую папку **на текущем диске** |
| `MKDIR "lessons"` | Создать папку |
| `KILL "old.bas"` | Удалить файл |
| `HELP COMMAND` | Встроенный справочник: например, `HELP CIRCLE` |
| `OPTION LIST` | Показать все текущие OPTION'ы |

## Хоткеи в REPL

Чтобы не печатать руками часто используемое:

- **F2** — `RUN`
- **F3** — `LIST`
- **F4** — `EDIT`
- **F10** — `AUTOSAVE` (приём программы по последовательному порту, нужно для XMODEM/копипаста с компа)
- **F11** / **F12** — `XMODEM RECEIVE` / `XMODEM SEND`

`F1` и `F5`–`F9` пустые, но их можно назначить через `OPTION FNKEY`.

В строке REPL:
- стрелки ←/→ — двигают курсор по строке;
- стрелки ↑/↓ — крутят историю команд;
- `Backspace` — удалить символ слева, `Delete` — справа от курсора;
- `Enter` — выполнить команду, `Ctrl-C` — прервать выполнение программы.

> Мануал PicoMite описывает ещё клавиши `Home`, `End`, `Insert` для редактирования строки — но клавиатура PicoCalc компактная, и не у всех этих клавиш есть прямые соответствия (часть доступна через `Fn`-комбо, часть может вообще не отрабатывать в REPL). Пробуй и смотри — на повседневное программирование это не влияет.

## Где хранятся программы: два «диска» и Flash-слоты

PicoMite видит **два логических диска**:

- `A:` — внутренняя флеш чипа, файловая система **LittleFS**. Маленькая, ~1.5 МБ, но всегда доступна, не зависит от SD.
- `B:` — карта **SD**, файловая система **FATFS**. Сколько вставил — столько и есть.

Текущий диск переключаешь как в DOS — буквой диска с двоеточием **как отдельной командой**:

```basic
> B:
> FILES
```

Это именно смена диска, не директории. `CHDIR` работает уже **внутри** текущего диска и не принимает префикс другого диска (`CHDIR "B:/..."` из `A:` даст `Error: only valid on current drive`). То есть рабочий паттерн — сначала `B:`, потом `CHDIR "lessons"`.

> 📌 **У каждого диска своя cwd, и она запоминается.** PicoMite держит две независимые «текущие папки» — одну для `A:`, одну для `B:` — и команды `A:` / `B:` лишь переключают, какая из них сейчас активна. То есть после `B:` → `CHDIR "lessons"` → `A:` → `B:` ты окажешься обратно в `B:/lessons`, а не в корне SD. Это поведение из DOS/CP/M, и оно удобно: можно держать на каждом диске «свою рабочую папку» и прыгать между ними одной буквой. Если нужен именно корень — `CHDIR "/"` после переключения на нужный диск.

Файлы программ — обычные текстовые `.bas`, читаются на компе любым редактором.

Помимо файловой системы есть ещё **Flash-слоты** — отдельная область флеш-памяти, куда можно положить сразу несколько программ и переключаться между ними без перезагрузки:

```basic
FLASH SAVE 1        ' сохранить текущую программу в слот 1
FLASH LOAD 2        ' загрузить программу из слота 2 в память
FLASH LIST          ' показать что в каких слотах
FLASH ERASE 4       ' очистить слот
FLASH RUN 1         ' загрузить и сразу запустить
```

> ⚠️ Слот **3** зарезервирован под `LIBRARY` (общие функции для всех программ) — туда не сохраняй обычные программы.

Слотов несколько (точное число — `FLASH LIST` покажет, у разных RC отличается). Это удобно: лежит «угадайка», «часы», «pong», ты по `FLASH RUN n` запускаешь любую за полсекунды.

## Редактор `EDIT`: первое знакомство

Набери `EDIT` (или нажми **F4**). Откроется полноэкранный редактор. Внизу — узкая статус-строка вида:

```
EDIT MODE     Ln: 1     Col: 1     INS
```

`Ln`/`Col` — позиция курсора, `INS`/`OVR` — режим вставки или замены (переключается клавишей **Insert**). Подсказок по хоткеям внизу не показывается — ключевые клавиши надо знать самому. Канон от автора PicoMite такой:

| Клавиша | Что делает |
|---------|------------|
| **F1** | **SAVE** — фиксирует текст редактора как программу в памяти и возвращает в обычный REPL с приглашением `>`. Сама программа в памяти, можешь запускать `RUN`, `LIST`, `SAVE "name.bas"`. |
| **F2** | **RUN** — то же самое, что F1, плюс сразу запускает программу. После её завершения (или `Ctrl-C`) ты тоже в REPL. |
| **Esc** | **Discard** — спрашивает `Exit and discard all changes (Y/N):` и выходит **с потерей** изменений, если ответить `Y`. Использовать только когда хочешь выкинуть набранное. |
| **F3** | Поиск по тексту (`FIND`). Повторить поиск — `Shift+F3`. |
| **F4** | Внутри редактора — **MARK** (выделить блок для копирования/вырезания). В REPL — глобальный шорткат `EDIT` (открыть редактор на текущей программе в памяти). |
| **F5** | Вставить из буфера (`PASTE`). |
| Стрелки | Движение курсора по тексту |
| **Backspace** / **Delete** | Удалить символ слева/справа |
| **Insert** | Переключить `INS` ↔ `OVR` |
| **Ctrl-Q** / **Ctrl-W** | Альтернативы для F1 / F2 (если терминал не пропускает Fn-клавиши). |

Запомни главное: **F1 и F2 одинаково сохраняют программу в память**, отличие в том, что F2 сразу её запускает. **Esc — это «выкинуть всё, что набрал»**, не «save & exit». Когда нужно сохранить и продолжить, не запуская программу — нажми **F1**, потом **F4** (заново открыть тот же буфер в редакторе).

И ещё одна важная деталь: F1/F2 сохраняют **в память**, не в файл. Чтобы получить файл `.bas` на диске, в REPL после выхода надо отдельно набрать `SAVE "name.bas"`.

> На PicoCalc клавиша **DEL** перенесена в специальный скан-код — в редакторе работает как привычный Delete; **ALT** тоже переназначен под особенности корпуса. Это сделано в прошивке, разбираться не надо.

## Первая программа

Заходи в редактор:

```
> EDIT
```

Набери:

```basic
' hello.bas - первая программа на PicoCalc
PRINT "Hello, PicoCalc!"
PRINT "Now: " + TIME$
PAUSE 1000
PRINT "Done."
```

> Все строки в `PRINT`/`INPUT` — на английском намеренно: встроенные шрифты PicoMite не содержат кириллицу, и попытка вывести `"Привет"` даст на LCD кашу из псевдографики. В **комментариях** (`'`) кириллица работает свободно — токенизатор их игнорирует. Подробнее — на [главной странице курса](index.md#как-пользоваться).

Теперь нажми **F1** — редактор закроется, программа уйдёт в память, ты вернёшься в REPL к приглашению `>`. Запусти её и сохрани в файл:

```
> RUN
Hello, PicoCalc!
Now: 14:23:05
Done.
> SAVE "hello.bas"
```

(Можно было сразу из редактора нажать **F2** вместо F1 — тогда программа сохранилась бы в память **и** запустилась без `RUN` вручную. Удобно для быстрого цикла «правка → запуск».)

Программа теперь и в памяти, и в файле `hello.bas` на текущем диске (после загрузки это `A:`). После перезагрузки её можно вернуть так:

```
> LOAD "hello.bas"
> RUN
```

А если положить тот же файл в слот 1:

```
> FLASH SAVE 1
> FLASH LIST
```

— то после перезагрузки её можно достать командой `FLASH LOAD 1` или сразу `FLASH RUN 1`.

## Автозапуск при включении

Если хочешь, чтобы при включении PicoCalc автоматически стартовала какая-то программа:

```basic
OPTION AUTORUN ON         ' запускать программу из основной памяти
OPTION AUTORUN n          ' запускать программу из слота n
OPTION AUTORUN OFF        ' выключить
```

Удобно для финального capstone-проекта в конце курса — превращает PicoCalc в специализированную «коробочку».

## Упражнение

### 1. Перенеси `hello.bas` с `A:` на `B:`

У тебя уже есть `A:/hello.bas` — мы только что его сохранили. Перенесём программу с внутренней флеши на SD-карточку, в папку `B:/lessons/`. Заодно увидим, как в MMBasic устроен «перенос» при отсутствии команды `MOVE`.

Подготовка папки на SD:

```
> B:
> MKDIR "lessons"
> CHDIR "lessons"
```

Теперь скопируй файл с внутренней флеши на SD одной командой:

```
> COPY "A:/hello.bas" TO "B:/lessons/hello.bas"
> FILES
```

`FILES` должен показать `hello.bas` в текущей папке (`B:/lessons/`). Запусти его прямо отсюда:

```
> LOAD "hello.bas"
> RUN
```

Если хочешь именно **перенести** (а не оставить копию на `A:`) — `MOVE` в PicoMite нет, переносить надо в два шага: `COPY` + `KILL`:

```
> KILL "A:/hello.bas"
```

`KILL` принимает абсолютный путь с префиксом любого диска — переключаться на `A:` для удаления не требуется.

> `RENAME old AS new` работает только **в пределах одного диска**: `RENAME "A:/x" AS "B:/x"` падает с `Error : Only valid on current drive`. Поэтому перенос между `A:` и `B:` — всегда `COPY` + `KILL`. И обрати внимание на ключевое слово: в MMBasic это `AS`, не `TO` (с `TO` будет `Error : Syntax`).

### 2. Вторая программа: мини-часы

Теперь напиши новую программу — но если ты сейчас просто наберёшь `EDIT`, редактор откроется на **уже загруженном `hello.bas`** (он лежит в памяти, а `EDIT` без аргумента редактирует именно её). Есть два чистых способа начать новый файл:

**Способ A.** Сначала очистить память, потом редактировать пустой буфер:

```
> NEW
> EDIT
```

`NEW` стирает программу из памяти (файлы на диске не трогает). После `EDIT` откроется пустой редактор — пиши с нуля.

**Способ B.** Открыть конкретный файл по имени — `EDIT` с аргументом сам подгружает указанный файл (или создаёт пустой, если файла нет), не затрагивая текущую программу в памяти:

```
> EDIT "clock.bas"
```

Любым способом — набери:

```basic
' clock.bas - простые часы, выход по Ctrl-C
DO
  PRINT TIME$
  PAUSE 1000
LOOP
```

> `DO ... LOOP` — обычный бесконечный цикл, его подробно разберём в уроке 02. Пока достаточно знать, что строки между `DO` и `LOOP` повторяются, пока программу не прервут.

Дальше — как обычно: **F2** для выхода, потом в REPL:

```
> SAVE "clock.bas"
> RUN
14:23:05
14:23:06
14:23:07
^C
>
```

Жми **Ctrl-C** чтобы остановить — это и есть тот самый универсальный способ прервать любой цикл.

### 3. Слоты и OPTION LIST

1. Загрузи `hello.bas` обратно в память (`LOAD "hello.bas"` в `B:/lessons` или `LOAD "A:/hello.bas"`) и положи в Flash-слот 1: `FLASH SAVE 1`. Перезагрузи PicoCalc и запусти из слота: `FLASH RUN 1`.
2. Набери `OPTION LIST` — обрати внимание на текущие настройки (разрешение, шрифт, autorun). Не меняй пока ничего, просто посмотри.

Когда уверенно двигаешься между REPL и редактором, помнишь куда сохранять и чем `A:` отличается от `B:` — переходи к **[Уроку 01 — Язык: тур по MMBasic](01-language-tour.md)**.
